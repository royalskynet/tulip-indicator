## 核心模型（v2，務必先懂）· Core Model

- **七機制 M1–M7**（不變骨架），各以當代代理量測，對自身歷史窗取**百分位 0–100**（泡沫極端＝高）。
- **加權**：M1 25% / M2 18% / M3 14% / M4 10% / M5 10% / M6 15% / M7 8%（2026 AI 情境）。
- **raw → display**：危險線 raw 70 映射到顯示 90。`display = raw*90/70 (raw≤70)`；`display = 90+(raw−70)/3 (raw>70)`。
- **三組摘要**：放大器(M4+M5+M7) / 領先引信(M1+M2+M3) / 確認(M6)。報告攤三組，不攤七格。
- **無 bonus**：v1.1 跨層共振加成已取消，百分位線性和即可重現「慢爬→脈衝」。
- **stateful**：每代理原始讀數＋百分位分存 `history/ti_history_v2.csv`；轉折/二階導由序列跨期算，不靠單次搜尋猜。

## 執行流程 · Execution（使用者說「跑一次鬱金香指標」「!ti」「今天 TI 多少」）

1. **逐代理抓數**：依 `indicators.yaml` 各代理 `fetch` 取最新值。
   - 先試 `primary_url`（直接 web_fetch，如 FRED `BAMLH0A0HYM2`）。
   - 空殼/JS 渲染/無數值 → 改 `search`，取前三筆最新具體數值。
   - 兩者皆失敗：該代理標「未驗證」、不計入機制（見下），列入報告未驗證欄。
   - **禁止用記憶舊值或推測補值。**
2. **代理 → 百分位**：對該代理自身歷史窗（市場日頻 504–756 日｜capex 季度 8–12 季｜流量 24–36 月）取百分位。
   水位類直接取值；轉折類先算 RoC/二階導再取百分位。自存不足時以外部序列 bootstrap 並旗標 `bootstrapped=true`。
3. **百分位 → 機制 → raw → 顯示**：機制 = 可得代理百分位均值；缺代理只對可得者平均並重正規化、旗標未驗證；
   缺整個機制 → 對可得權重重新正規化。raw = Σ(w_i·M_i)，再做 display 映射。可直接用 `score.py` 計（填 `PROXIES` 百分位後 `python3 score.py`）。
4. **寫入歷史**：追加一列到 `history/ti_history_v2.csv`（格式見表頭：date, TI_display, TI_raw, state, M1–M7, 三組, 各代理百分位, unverified, note）。
5. **輸出報告**（固定格式見下）。

## 區間對照 · Bands（顯示刻度）

| TI | 狀態 | 建議 |
|---|---|---|
| 0–50 | 平靜 | 正常配置，每月查 |
| 50–75 | 累積 | 停止加碼風險，壓測組合能否扛 −35% 不被迫賣 |
| 75–90 | 警戒/逐步退場 | **逐步退場窗口，90 前退完**；改週查 M1/M2/M6 |
| 90–100 | 進行中/太晚 | 控損，不在底部被迫賣，保留子彈 |

**跳機紀律**：不要把跳傘設在 90。理想 75 起分批退、90 前退完；90–100 是脈衝描述線，看到就太晚。

## 個別代理備注 · Proxy Notes

- **M1 BDC（折價＋贖回閘門）**：折價百分位之外，若出現**贖回閘門（gate）**事件，視為單一強火星，分析欄明確點出。
- **M2 capex（背離訊號）**：慢錨（季度 capex YoY 二階導 × capex/OCF 百分位）與快代理（AI 債發行/利差、GPU 租價）**背離 ＝ 最強頂部訊號**，報告必標。
- **M3 IPO**：「流標/撤回率」留 M3（買家枯竭）；「首日暴漲 FOMO」屬 M7（買家過剩），**勿放 M3，方向相反會抵銷**。
- **M7 情緒**：最吵、最晚、最低權重。獨立成格收容純情緒（IPO 首日狂歡、開戶潮、0DTE），勿讓它寄生扭曲其他機制。

## 判讀紀律 · Discipline（嚴格遵守）

- **狀態遷移需連續兩期確認**：單期跳區先標「待確認」。
- **不得給具體日期預測**：時間框架只以區間 + 歷史類比（n≤5）表述。
- **不得因使用者倉位或情緒調整任何代理分數**：分數是分數，建議是建議。
- **不為過去擬合**：2000/2008 只當否證抓壞代理；某代理在「已知是該類型泡沫」期間完全沒動 → 回去修代理，不是調權重。
- 使用者質疑閾值/權重：討論可以，修改需寫入 `CHANGELOG.md` 並升版本。
- 使用者有 FIRE 長期規劃背景：最高原則「不被迫在底部賣出」，不鼓勵全進全出式擇時。

## 頻率建議 · Cadence
- TI < 50：每月一次
- TI 50–75：領先引信（M1/M2/M3）每週，全表每月
- TI > 75：全表每週，M6 可每日

# CLAUDE.md — 鬱金香指標執行指令

你（Claude，任何型號含 Sonnet）被要求執行「鬱金香指標（TI）」查核時，依本文件操作。
完整方法論見 METHODOLOGY.md，指標機讀規格見 indicators.yaml。

## 與使用者的溝通語言
繁體中文。風格：直接、結構化、不灌水、不過度安撫。
數據不確定就明說「未驗證」，禁止編造數值。

## 執行流程（使用者說「跑一次鬱金香指標」「今天 TI 多少」等）

1. **逐項抓數**：依 indicators.yaml 的 `fetch` 欄位取得每個指標最新值。
   - 優先試 `primary_url`（直接 web_fetch）。
   - 若頁面為空殼（JS 渲染）或無數值，改用 `search_query` 搜尋。
   - 兩者皆失敗：計 0 分、標註「未驗證」，並在報告中列出。
   - 禁止用記憶中的舊值或推測值代替即時數據。
2. **計分**：依 METHODOLOGY.md 第 1–2 節規則給每項 0/1/2，算四層分數、
   組合加成、TI 終值。
3. **寫入歷史**：將本次結果追加一列到 history/ti_history.csv（格式見該檔表頭）。
4. **輸出報告**，固定格式如下：

```
## 鬱金香指標 | YYYY-MM-DD
TI = XX（前次 XX，YYYY-MM-DD）　狀態：[平靜/累積期/引信備妥/點火/進行中]

| 層 | 分數 | 亮燈項 |
|---|---|---|
| A 倉位 | xx | A1🔴 A4🟡 ... |
| B 邊緣流動性 | xx | ... |
| C 信貸 | xx | ... |
| D 結構確認 | xx | ... |
組合加成：+x（觸發條件）
未驗證項目：...

### 分析（3–6 句）
- 與前次比較：哪些指標變動、方向
- 當前最值得盯的單一斷點
- 依第 3 節區間表給出條件時間框架，必須附帶「歷史類比 n≤5」免責
### 建議動作
依 METHODOLOGY.md 第 3 節行動建議欄，對應當前區間給一條，不多給。
```

## 抓取失敗處理層級

1. **先試 `primary_url`**：直接 web_fetch。
2. **若頁面為空殼（JS 渲染）或無具體數值**：改用 `search_query` 搜尋，取前三筆結果中最新的具體數值。
3. **若搜尋結果也無具體數字**：記 0 分、標註「未驗證」，在報告「未驗證項目」欄列出。不得用記憶中的舊值補填。
4. **連續兩期未驗證的指標**：在報告中標註「建議替換來源」，並在分析欄說明。

## 個別指標執行備注

**A3（BofA 現金水位）**：觸發黃燈（<4%）時，需額外判斷方向——「從高位下降至 <4%」
（機構加倉，訊號弱）vs「已在低位後回升」（機構開始防禦，訊號強）。
若為前者，黃燈保留但在分析欄標註「方向偏多，訊號強度折半」。

**A5（台股融資餘額）**：先抓本週數值，再搜尋上週數值比較。
若上週數值搜尋不到，黃燈可認定但不升紅，並在報告標註「待下週確認是否連兩週」。

**B4（台股全民化）**：優先試 TWSE API（當沖數據），URL 格式見 indicators.yaml。
若 API 返回空白，改用搜尋。新開戶數無 API，一律以搜尋集保結算所月報為主。

## 判讀紀律（嚴格遵守）

- **狀態遷移需連續兩期確認**：單期跳區間先標「待確認」。
- **不得給出具體日期預測**。時間框架只能以區間 + 歷史類比表述。
- **不得因使用者倉位或情緒調整分數**。分數是分數，建議是建議。
- 若使用者質疑某指標閾值，討論可以，修改需寫入 CHANGELOG.md 並升版本。
- 使用者有 FIRE 長期規劃背景：建議動作以「不被迫在底部賣出」為最高原則，
  不鼓勵全進全出式擇時。

## 頻率建議
- TI < 40：每月一次
- TI 40–60：B、C 層每週，全表每月
- TI > 60：全表每週，D 層可每日
